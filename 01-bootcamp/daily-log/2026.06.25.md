## 학습 요약
AI 시대에는 모델을 직접 처음부터 만드는 것보다, 이미 공개된 모델과 도구를 이해하고 가져와서 목적에 맞게 조합하는 능력이 중요하다.

## 본 내용
### LLM 개념
방대한 양의 텍스트 데이터를 기반으로 학습된 인공지능 모델.

하지만, 요즘 ChatGPT 같은 모델은 단순히 텍스트만 다루는 LLM을 넘어서고 있다. 

같은 여러 입력을 처리하므로, 최근에는 멀티모달 모델 또는 Large Multimodal Model에 가까워졌다고 볼 수 있다.

### GAN
Generator와 Discriminator가 경쟁하면서 학습하는 생성 모델이다.
| 구성            | 역할                                 |
| ------------- | ---------------------------------- |
| Generator     | 가짜 데이터를 생성                         |
| Discriminator | 진짜와 가짜를 구분                         |
| 학습 방식         | 생성자는 더 진짜처럼 만들고, 판별자는 더 잘 구분하도록 경쟁 |

#### 비유
- Generator = 지폐 위조범
- Discriminator = 위조지폐를 잡는 경찰
- 예시) StyleGAN, Pix2Pix, CycleGAN

### Diffusion 모델
1) 원본 이미지에 노이즈를 점점 추가한다.
2) 완전한 노이즈 상태로 만든다.
3) 그 노이즈에서 다시 원본에 가까운 이미지를 복원하는 과정을 학습한다.

# 실습
## Stable Diffusion
| 유형             | 의미                                 |
| -------------- | ---------------------------------- |
| Image-to-Image | 기존 이미지를 입력하고 스타일이나 내용을 바꿔 새 이미지 생성 |
| Text-to-Image  | 텍스트 프롬프트만으로 이미지 생성                 |

## ControlNet(강의 중 실습 안함)
Diffusion 모델을 더 세밀하게 제어하기 위한 기술이다.

## AlexNet
이미지 분류용 CNN 모델이다.

Convolution → ReLU → Pooling → Fully Connected → Classification

| 구성                       | 역할                     |
| ------------------------ | ---------------------- |
| Convolution              | 선, 모서리, 패턴 같은 특징 추출    |
| ReLU                     | 중요한 신호를 살리는 활성화 함수     |
| Max Pooling              | 이미지 크기를 줄이면서 중요한 특징 유지 |
| Linear / Fully Connected | 최종 클래스 분류              |

### Data Augmentation
학습 이미지를 일부러 변형해서 데이터 다양성을 늘리는 방법이다.

예시 : 
| 방법              | 설명                        |
| --------------- | ------------------------- |
| Random Crop     | 이미지를 임의로 잘라냄              |
| Horizontal Flip | 좌우 반전                     |
| Resize          | 크기 변경                     |
| Normalize       | 값 범위 정규화                  |
| Tensor 변환       | PyTorch가 계산 가능한 숫자 형태로 변환 |

### Data Loader
데이터를 모델에 넣기 좋게 나눠주는 역할이다.

전체 데이터셋 → batch 단위로 나눠서 모델에 공급

| 데이터        | 역할              |
| ---------- | --------------- |
| Train Data | 모델이 공부하는 문제집    |
| Test Data  | 모델 성능을 확인하는 시험지 |

### Loss Function
분류 모델에서는 보통 Cross Entropy Loss를 사용한다. 

Loss가 작아질수록 정답에 가까워진다.

### Optimizer
loss를 줄이기 위해 모델의 가중치를 수정하는 역할이다.

| 개념            | 의미                       |
| ------------- | ------------------------ |
| Learning Rate | 한 번에 얼마나 크게 수정할지         |
| Momentum      | 학습 방향을 어느 정도 유지할지        |
| Weight Decay  | 가중치가 너무 커지는 것을 막아 과적합 방지 |

이 실습에서는 SGD를 쓰지만, 요즘은 Adam을 많이 이용한다.

### Train & Test
- 예측한다
- 얼마나 틀렸는지 loss를 계산한다
- 거꾸로 오차를 전파한다
- 가중치를 수정한다
- 다시 예측한다

Train 데이터 뿐만 아니라 Test 데이터에서도 잘 맞출 수 있어야 한다. (그렇지 않으면 과적합이다.)

### Checkpoint
학습 중간 저장 파일이다.

| 저장 대상           | 의미            |
| --------------- | ------------- |
| epoch           | 몇 번째 학습까지 했는지 |
| model state     | 모델 가중치        |
| optimizer state | optimizer 상태  |
