## 학습 요약
AI 모델을 쓰기 전에는 데이터를 숫자로 정리하고, 결측치·형식·변수 관계를 이해한 뒤, 분석 결과를 해석할 수 있어야한다.

## 본 내용
### 빅데이터 개념
데이터의 3가지 분류
| 구분      | 의미                           | 예시                 |
| ------- | ---------------------------- | ------------------ |
| 정형 데이터  | 표처럼 규칙적으로 정리된 데이터            | CSV, 엑셀, DB 테이블    |
| 반정형 데이터 | 구조는 있지만 표처럼 완전히 고정되지는 않은 데이터 | JSON, XML, HTML    |
| 비정형 데이터 | 고정된 구조가 없는 데이터               | 이미지, 영상, 음성, SNS 글 |

빅데이터의 5V(Volume, Velocity, Variety + Veracity, Value)
| 개념       | 뜻         | 핵심 질문         |
| -------- | --------- | ------------- |
| Volume   | 데이터의 양    | 얼마나 많은가?      |
| Velocity | 데이터 생성 속도 | 얼마나 빠르게 생기는가? |
| Variety  | 데이터 다양성   | 어떤 형태인가?      |
| Veracity | 데이터 정확성   | 믿을 수 있는가?     |
| Value    | 데이터 가치    | 분석해서 무엇을 얻는가? |

### 빅데이터 처리 단계
데이터 생성 → 수집 → 저장 → 분석 → 표현


# 실습
## 와인 품질 예측
와인의 화학적 속성을 독립 변수로 두고, 와인의 품질 점수 quality를 종속 변수로 두어 품질 등급을 예측하는 것
### 실습 흐름
- red wine CSV 읽기
- white wine CSV 읽기
- 각 데이터에 type 컬럼 추가
- red + white 데이터 병합
- 컬럼명 공백을 _로 변경
- 기술통계 확인
- red/white 품질 평균 비교
- t-검정 수행
- 선형 회귀분석 수행
- 예측값 출력
- 시각화

### 코드 결과
| 항목             | 결과           |
| -------------- | ------------ |
| 레드 와인 데이터 수    | 1,599개       |
| 화이트 와인 데이터 수   | 4,898개       |
| 전체 데이터 수       | 6,497개       |
| 레드 와인 평균 품질    | 약 5.636      |
| 화이트 와인 평균 품질   | 약 5.878      |
| t-검정 p-value   | 약 `8.17e-24` |
| 회귀분석 R-squared | 약 0.292      |

#### 해석
화이트 와인의 평균 품질 점수가 레드 와인보다 높고, p-value가 0.05보다 훨씬 작으므로 이 차이는 통계적으로 유의미하다.

다만 R-squared가 0.292라는 것은, 이 회귀모델이 와인 품질 변동을 약 29.2% 정도 설명한다는 뜻입니다. 아주 높은 설명력은 아니지만, 화학 성분만으로 품질을 예측했다는 점을 고려하면 기초 실습용으로는 의미가 있다.


## 반드시 이해해야 할 개념
| 개념             | 설명                       |
| -------------- | ------------------------ |
| 독립 변수          | 결과에 영향을 주는 입력값           |
| 종속 변수          | 예측하려는 결과값                |
| t-검정           | 두 집단 평균 차이가 우연인지 아닌지 판단  |
| p-value        | 결과가 우연히 나왔을 가능성          |
| 회귀분석           | 입력 변수로 연속적인 결과값을 예측      |
| 계수 coefficient | 각 변수가 결과에 주는 영향의 방향과 크기  |
| R-squared      | 모델이 결과를 얼마나 설명하는지 나타내는 값 |


## 타이타닉 생존률 분석
### 목표 
생존 여부와 관련된 변수들이 무엇인지 분석.
### 실습 흐름
- seaborn에서 titanic 데이터셋 불러오기
- CSV로 저장
- 결측치 확인
- age 결측치 처리
- embarked, embark_town 결측치 처리
- deck 결측치 처리
- 생존자/사망자 수 확인
- 성별 생존률 시각화
- 객실 등급별 생존률 시각화
- 피어슨 상관계수 계산
- 히트맵 시각화

### 코드 결과
| 항목                    | 결과            |
| --------------------- | ------------- |
| 전체 승객 수               | 891명          |
| 사망자                   | 549명          |
| 생존자                   | 342명          |
| 주요 영향 변수              | 성별, 객실 등급, 요금 |
| `adult_male`와 생존 상관계수 | 약 -0.557      |
| `pclass`와 생존 상관계수     | 약 -0.338      |
| `fare`와 생존 상관계수       | 약 +0.257      |

#### 해석
| 변수                 | 해석                                     |
| ------------------ | -------------------------------------- |
| `adult_male` 음의 상관 | 성인 남성일수록 생존 확률이 낮은 경향                  |
| `pclass` 음의 상관     | 객실 등급 숫자가 낮을수록, 즉 1등급에 가까울수록 생존 확률이 높음 |
| `fare` 양의 상관       | 요금을 많이 낸 승객일수록 생존 확률이 높은 경향            |
| `age` 약한 음의 상관     | 나이가 어릴수록 생존 확률이 약간 높은 경향               |


## 크롤링 실습
### 크롤링 개념
웹에서 데이터를 수집하는 작업. 웹사이트에서 데이터를 추출하는 프로그램을 크롤러 또는 스파이더라고 부른다.

이 실습에서는 일반적인 HTML 크롤링이 아니라 네이버 Open API를 이용한 데이터 수집했다.

요청 URL 생성 → 네이버 API 호출 → JSON 응답 받기 → 필요한 정보 저장
