# 학습 요약
본격적인 코딩 수업이라기보다, 앞으로 수업에서 어떤 관점으로 AI 서비스를 만들 것인지 설명하는 오리엔테이션 + 개발 관점 소개


### 강의 전체 목적
“AI 모델 자체를 깊게 수학적으로 만드는 수업”이 아니라, 이미 존재하는 AI 모델과 도구를 활용해서 실제 서비스를 기획·개발하는 실무형 수업

### 수업 운영 방향
- 초반 차시 : Python, 개발 환경, 이미지/언어 모델 개념
- 중반 차시 : 이미지 모델, LLM, 모델 활용 방식
- 후반 차시 : Gemini 등을 활용한 프로젝트 실습
- 평가 : 중간고사와 기말고사가 있음

### 개발도구
- Visual Studio Code
- Anaconda
- Jupyter Notebook
- AI service ex) Gemini, ect ...


## 본 내용

### 1. 바이브 코딩

개발자가 모든 코드를 직접 작성하는 대신,
만들고 싶은 기능과 구조를 정확히 설명하고,
AI가 생성한 코드를 검토·수정·조합하면서 개발하는 방식

개발자 + 기획자가 되어야 함.

### 2. 프롬프트

대충 묻지 말고, 프로그램을 설계하듯이 논리적으로 쪼개서 물어봐야 한다.
맞춤법, 용어, 논리 구조를 정확히 해서 질문해야 AI가 좋은 답을 준다.

### 3. 데이터에서 중요한 것

데이터를 무조건 많이 모으는 것보다, 지금은 데이터 품질과 라벨링이 더 중요하다.

### 4. 개발자에게 필요한 역량 변화

문제 정의 능력, 용어(정의) 이해, 아키텍처 이해, 프롬프트 설계, 정책/윤리 이해, 폭넓은 지식
